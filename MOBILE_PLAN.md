# 移动端适配计划

## 约束
- **必须纯手机浏览器可玩，禁止依赖电脑键盘**
- 所有交互必须通过触摸屏完成
- 界面布局需适配竖屏手机（375px-430px 宽度）

## 当前状态
- 未发现键盘或触摸事件绑定
- 需要检查实际交互方式（可能使用点击/按钮，需要验证）

## 修改方案

### 1. 全面检查交互方式
- 逐一检查所有 JS 文件，确认当前用户输入方式
- 如果已经是点击式交互，重点做触摸优化
- 如果有隐藏的键盘依赖，全部替换为触摸

### 2. 触摸友好设计
- 所有可点击元素最小 44px × 44px
- 按钮间距至少 12px
- 文字链接区域不小于 44px 高度
- 添加 :active 伪类视觉反馈

### 3. 竖屏适配
- 主容器宽度 max-width: 100vw
- 使用 flexbox/grid 自适应布局
- 图片和卡片不超出屏幕宽度
- 横屏时给提示建议竖屏游玩（如适用）

### 4. 浏览器兼容
- `touch-action: manipulation`
- `-webkit-touch-callout: none`
- `overscroll-behavior: none`
- viewport 确认 `user-scalable=no, viewport-fit=cover`
